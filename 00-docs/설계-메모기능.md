# 사색노트 분리 + 메모(inbox) 기능 — 설계 확정본

grilling 세션으로 합의된 설계. 구현 중 판단이 갈리면 이 문서가 기준.

## 왜 이렇게 하는가 (합의된 전제)

- 메모는 **비공개 원재료 창고**. 공개 스트림이 아니다. 조각 대부분은 영원히 글이 되지 않아도 된다.
- 생각은 **컴퓨터 밖(폰)** 에서 떠오른다. 그래서 폰에서 쓸 수 있어야 한다.
- 이 기능은 기능 부족이 아니라 **쓰는 순간의 마찰** 때문에 죽는다. 마찰을 늘리는 제안은 기본적으로 거절한다.

## 구조

- 저장소 2개: `citron0137.github.io`(포트폴리오, public) + **`app-musings`(사색노트+메모, public)**
- 도메인: **`musings.rahoon.site`** (CNAME 파일 + DNS CNAME → citron0137.github.io)
- `app-musings`는 **완전 자립**. 본체의 css/js/layout 을 참조하지 않는다 (도메인이 달라 성립도 안 함)
- 사색노트 히스토리는 `git subtree split` 으로 보존 이전
- 본체 히스토리 세탁 안 함. 옛 URL(`rahoon.site/musings/*`)은 **404 로 둔다** (리다이렉트 안 만듦)

## 메모의 형태

- **본문 텍스트 하나뿐.** 제목·태그·요약 필드 없음. 태그가 필요하면 본문에 `#상담` 처럼 쓰고 읽을 때 파싱.
- **상태 없음.** 삭제·편집·'처리됨' 표시 전부 없음. 받은편지함이 아니라 퇴적층.
  - 상태를 만들면 "밀린 메모 87개"가 되고, 그 순간 부채가 되어 안 쓰게 된다.
- 메모 하나 = 암호문 파일 하나. `inbox/2026-07-12-034117.enc` (KST 기준 파일명)

## 저장 경로

- 폰(PWA) → **GitHub API 직접 커밋** → `app-musings` 의 **고아(orphan) `inbox` 브랜치**
  - `main` 과 공통 조상 없음 → merge/rebase 영원히 없음. 사이트 재배포도 안 일어남
- 토큰: fine-grained PAT, **`app-musings` 저장소 Contents 읽기/쓰기만**
  - **`token.enc` 로 암호화해서 저장소에 커밋한다** (사이트에 공개 서빙됨). 폰에는 토큰을 저장하지 않는다
  - **등록도 페이지에서 한다** — 받은 토큰으로 페이지가 스스로 `token.enc` 를 커밋(부트스트랩)
  - 어느 기기에서든 비밀번호만 알면 쓸 수 있다. 폰을 잃어버려도 거기서 나올 것이 없다
  - **대가**: 암호문이 공개되므로, 비밀번호가 깨지면 토큰도 깨진다.
    그 위험을 **90일 만료**로 제한하는 것이 유일한 안전장치다 — 이건 옵션이 아니라 필수 부품
  - `encrypt-token.js` 가 90일 초과 만료를 거부하고, 페이지가 D-14 부터 만료 경고를 띄운다
  - 비밀번호는 zxcvbn 4/4, 패턴 미탐지(bruteforce only). 이 전제가 깨지면 이 설계도 깨진다
- 암호화: 기존 잠긴 노트와 **같은 비밀번호**, AES-256-GCM + PBKDF2-SHA256
  - 반복수 **60만** (`_scripts/lib/musings-crypto.js` 의 `ITERATIONS` 한 곳에만 있음)
  - `.enc` v2 포맷: `{"v":2,"iter":600000,"iv":"<b64>","tag":"<b64>","data":"<b64>"}`
  - `iter` 를 파일에 기록하므로 앞으로 파라미터를 또 올려도 옛 파일이 열린다 (v1 = iter 없음 → 100000)

## 쓰는 순간 / 읽는 순간

- 페이지를 열면 **커서가 곧바로 텍스트 영역에.** 비번은 **저장 버튼을 누른 뒤** 묻는다.
  - 인증이 생각을 가로막으면 안 된다. 새벽 세 시의 생각은 비밀번호 화면 앞에서 증발한다.
- **서비스워커 + 홈 화면 앱(PWA).** 비행기모드에서도 열린다
  - 전송 실패 → **암호문 상태로 큐**에 쌓고, 신호 잡히면 자동 전송. "대기 중 N개" 표시
  - 평문은 **작성 중에만** 폰에 존재. 저장/큐잉되는 순간 삭제
- 폰 뷰어는 **확인용**(최근 것만). 전체 아카이브 화면은 만들지 않는다
  - 쌓인 더미를 사람이 마주하지 않게 하는 것이 의도. 정리는 노트북에서 Claude 가 읽고 주제별 목차로 내놓는다

## 받아들인 대가 (다시 논의하지 말 것)

- 메모 **작성 시각은 커밋 히스토리에 공개**되고, 암호문도 public 저장소에서 누구나 받아갈 수 있다
- 폰 토큰 + 비밀번호가 함께 뚫리면 musings 사이트 변조 가능 (포트폴리오 본체는 안전)
- private 저장소로 하면 위 둘이 사라지지만, Pro 가 곧 끝나므로 public 으로 간다

### 외부 CDN 을 쓰지 않는 이유

`/inbox/` 는 **마스터 비밀번호를 입력받는 페이지**다. 서드파티 스크립트(예: Tailwind CDN)가 하나라도
끼면, 그 CDN 이 침해되는 순간 비밀번호가 그대로 새어 나간다. 다른 페이지에서는 스타일링일 뿐이지만
여기서는 다르다. 그래서 스타일은 전부 인라인이고, 스크립트는 같은 오리진의 `/js/musings-crypto.js`
하나뿐이다. 덤으로 오프라인에서도 화면이 깨지지 않는다. **이 페이지에 외부 리소스를 추가하지 말 것.**

## 토큰 등록 (90일마다 반복) — 폰에서 끝난다

1. GitHub 에서 fine-grained PAT 발급
   - Repository access: **`citron0137/app-musings` 만**
   - Permissions: **Contents = Read and write** (그 외 전부 No access)
   - Expiration: **90일 이하** ← 이 설계의 유일한 안전장치
   - classic 토큰(`ghp_`) 금지 — 모든 공개 저장소에 쓰기 권한이 열린다
2. 폰에서 `musings.rahoon.site/inbox/` 를 연다
3. 아무거나 적고 **저장** → 비밀번호 → **토큰 등록 화면**이 뜬다 → 토큰 + 만료일 입력
4. 페이지가 토큰을 비밀번호로 암호화해서 **자기 자신이 `token.enc` 를 커밋**한다

**노트북이 필요 없다.** 이게 중요한 이유: 만료가 하필 출장 중이나 지친 날에 오면,
노트북 앞에 갈 때까지 토큰이 방치되고 메모는 큐에만 쌓인다. 안전장치가 기능을 죽이는 구조였다.

페이지도 **90일 상한을 강제**한다(날짜 입력의 min/max + 제출 시 재검증). 스크립트만 막고
페이지를 안 막으면 안전장치에 구멍이 난다.

만료 **14일 전부터** 빨간 배너로 알린다. 만료된 채로 두면 메모가 큐에만 쌓이지만
**글이 사라지지는 않는다** — 토큰을 갱신하면 밀린 것이 한꺼번에 전송된다.

### 노트북에서 하는 방법 (대안)

```bash
notepad _locked/token.txt                              # 평문 토큰 (gitignore)
node _scripts/encrypt-token.js --expires YYYY-MM-DD    # → token.enc
git add token.enc && git commit -m "token.enc 갱신" && git push
```

### 사고 기록 — `_inbox/` 가 gitignore 되지 않고 있었다

`pull-inbox.js` 가 내려놓는 **메모 평문**이 미추적 파일로 노출돼 있었다.
`git add -A` 한 번이면 public 저장소에 평문이 올라갈 뻔했다. `.gitignore` 에 추가해 막음.
**평문을 만들어내는 스크립트를 추가할 때는 반드시 gitignore 를 함께 확인할 것.**


### 사고 기록 — 2단계에서 잡은 버그 셋

1. **오프라인에서 글이 날아갈 뻔했다.** `save()` 가 토큰을 먼저 가져오려 해서, 신호가 없으면
   암호화도 큐잉도 못 하고 실패했다. 서비스워커가 `token.enc` 를 캐시하도록 고쳐서
   오프라인에서도 비번 검증 → 암호화 → 큐 적재가 되게 했다. 지하철 때문에 폰 캡처를 택했는데
   정작 지하철에서 안 되는 물건이 될 뻔했다.

2. **서비스워커 설치가 통째로 실패했다.** `install` 의 `cache.addAll` 에 `/token.enc` 가 들어
   있었는데, `addAll` 은 하나라도 404 면 전체가 실패한다. 즉 토큰 등록 전에는 오프라인 기능이
   아예 없었다. 프리캐시에서 빼고, 성공 응답일 때만 캐시하도록 고쳤다(404 캐싱도 막음).
   **`token.enc` 를 지워보자는 요청이 없었다면 영영 안 드러났을 버그다.**

3. **오프바이원 — 스스로 채운 값을 스스로 거부했다.** 만료일을 날짜로 저장하는데 남은 날짜를
   그 날의 끝(23:59:59)까지 올림해서 세는 바람에, 정확히 90일 뒤인 기본값이 91일로 계산됐다.
   KST 달력일 차이로 고쳤다. 같은 버그로 만료 당일 아침에 "이미 만료"로 튕겼을 것이다.

**교훈: 캐시·날짜 경계·오프라인은 "정상 경로"만 테스트하면 전부 통과한다.**
