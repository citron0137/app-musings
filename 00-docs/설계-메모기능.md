# 사색노트 분리 + 메모(inbox) 기능 — 설계 확정본

grilling 세션으로 합의된 설계. 구현 중 판단이 갈리면 이 문서가 기준.

## 왜 이렇게 하는가 (합의된 전제)

- 메모는 **비공개 원재료 창고**. 공개 스트림이 아니다. 조각 대부분은 영원히 글이 되지 않아도 된다.
- 생각은 **컴퓨터 밖(폰)** 에서 떠오른다. 그래서 폰에서 쓸 수 있어야 한다.
- 이 기능은 기능 부족이 아니라 **쓰는 순간의 마찰** 때문에 죽는다. 마찰을 늘리는 제안은 기본적으로 거절한다.

## 구조

- 저장소 2개: `citron0137.github.io`(포트폴리오, public) + **`app-musings`(사색노트+메모, public)**
- 도메인: **`musings.rahoon.site`** (CNAME 파일 + DNS CNAME → citron0137.github.io)
- `app-musings`는 **완전 자립**. 본체의 css/js/layout 을 참조하지 않는다 (도메인이 달라 성립도 안 함)
- 사색노트 히스토리는 `git subtree split` 으로 보존 이전
- 본체 히스토리 세탁 안 함. 옛 URL(`rahoon.site/musings/*`)은 **404 로 둔다** (리다이렉트 안 만듦)

## 메모의 형태

- **본문 텍스트 하나뿐.** 제목·태그·요약 필드 없음. 태그가 필요하면 본문에 `#상담` 처럼 쓰고 읽을 때 파싱.
- **상태 없음.** 삭제·편집·'처리됨' 표시 전부 없음. 받은편지함이 아니라 퇴적층.
  - 상태를 만들면 "밀린 메모 87개"가 되고, 그 순간 부채가 되어 안 쓰게 된다.
- 메모 하나 = 암호문 파일 하나. `inbox/2026-07-12-034117.enc` (KST 기준 파일명)

## 저장 경로

- 폰(PWA) → **GitHub API 직접 커밋** → `app-musings` 의 **고아(orphan) `inbox` 브랜치**
  - `main` 과 공통 조상 없음 → merge/rebase 영원히 없음. 사이트 재배포도 안 일어남
- 토큰: fine-grained PAT, **`app-musings` 저장소 Contents 읽기/쓰기만**
  - 폰 localStorage 에 **비밀번호로 암호화해서** 보관. 비번 없이는 토큰도 메모도 안 열림
  - 만료 시 폰에서 재등록 (1년 주기)
- 암호화: 기존 잠긴 노트와 **같은 비밀번호**, AES-256-GCM + PBKDF2-SHA256
  - 반복수 **60만** (`_scripts/lib/musings-crypto.js` 의 `ITERATIONS` 한 곳에만 있음)
  - `.enc` v2 포맷: `{"v":2,"iter":600000,"iv":"<b64>","tag":"<b64>","data":"<b64>"}`
  - `iter` 를 파일에 기록하므로 앞으로 파라미터를 또 올려도 옛 파일이 열린다 (v1 = iter 없음 → 100000)

## 쓰는 순간 / 읽는 순간

- 페이지를 열면 **커서가 곧바로 텍스트 영역에.** 비번은 **저장 버튼을 누른 뒤** 묻는다.
  - 인증이 생각을 가로막으면 안 된다. 새벽 세 시의 생각은 비밀번호 화면 앞에서 증발한다.
- **서비스워커 + 홈 화면 앱(PWA).** 비행기모드에서도 열린다
  - 전송 실패 → **암호문 상태로 큐**에 쌓고, 신호 잡히면 자동 전송. "대기 중 N개" 표시
  - 평문은 **작성 중에만** 폰에 존재. 저장/큐잉되는 순간 삭제
- 폰 뷰어는 **확인용**(최근 것만). 전체 아카이브 화면은 만들지 않는다
  - 쌓인 더미를 사람이 마주하지 않게 하는 것이 의도. 정리는 노트북에서 Claude 가 읽고 주제별 목차로 내놓는다

## 받아들인 대가 (다시 논의하지 말 것)

- 메모 **작성 시각은 커밋 히스토리에 공개**되고, 암호문도 public 저장소에서 누구나 받아갈 수 있다
- 폰 토큰 + 비밀번호가 함께 뚫리면 musings 사이트 변조 가능 (포트폴리오 본체는 안전)
- private 저장소로 하면 위 둘이 사라지지만, Pro 가 곧 끝나므로 public 으로 간다

최초 `subtree split` 을 **`git fetch` 없이 낡은 로컬 클론에서** 실행했다. 로컬이 21커밋 뒤처져 있어서
